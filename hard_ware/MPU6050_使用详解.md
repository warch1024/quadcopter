# MPU6050 使用详解

## 一、传感器概览

MPU6050 是 InvenSense 公司推出的全球首款整合性 6 轴运动处理组件，内部集成了：

- **3 轴 MEMS 加速度计**：测量物体受到的加速度（包括重力），单位 g
- **3 轴 MEMS 陀螺仪**：测量物体旋转的角速度，单位 °/s
- **内置 DMP（数字运动处理器）**：可硬件直接输出四元数（本驱动未使用，采用软件 Mahony 算法）

### 坐标系定义

```
           +Y (前)
            ↑
            |
   -X ←-----+-----→ +X (右)
            |
            ↓
           +Z (下)
```

- **加速度计**：静止时 Z 轴读数为 +1g，X/Y 约为 0
- **陀螺仪**：正方向遵循右手定则

## 二、硬件连接

### I2C1 接口（本工程的接线）

```
STM32 F103            MPU6050
  PB6 (SCL)   ─────── SCL
  PB7 (SDA)   ─────── SDA
  3.3V        ─────── VCC
  GND         ─────── GND
```

- **I2C 地址**：7 位地址 `0x68`（AD0 引脚接 GND）；如果 AD0 接 3.3V，地址变为 `0x69`
- **上拉电阻**：模块通常自带 4.7kΩ，裸片需要外部加
- **供电电压**：2.375V ~ 3.46V，**不能接 5V**
- **VLOGIC**：可选，若不接则内部默认 3.3V

### 与 VL53L1X 共用 I2C 总线

MPU6050 使用 I2C1（PB6/PB7），VL53L1X 使用 I2C2（PB10/PB11），两条独立的 I2C 总线互不干扰。

## 三、寄存器操作

MPU6050 的寄存器地址为 **8 位**（不同于 VL53L1X 的 16 位地址）。

### 写时序

```
START + 0xD0(写) + REG_ADDR + DATA + STOP
```

### 读时序

```
START + 0xD0(写) + REG_ADDR + 重复START + 0xD1(读) + DATA + STOP
```

### 多字节读

MPU6050 支持自动地址递增，从起始地址连续读取 N 个字节。本驱动的 `i2c1_read_bytes()` 实现了此功能。

## 四、关键寄存器

| 地址 | 名称 | 说明 |
|------|------|------|
| 0x75 | WHO_AM_I | 固定值 0x68，用于验证设备身份 |
| 0x6B | PWR_MGMT_1 | 电源管理，写 0x00 唤醒设备 |
| 0x19 | SMPRT_DIV | 采样率分频器 |
| 0x1A | CONFIG | DLPF 低通滤波器配置 |
| 0x1B | GYRO_CONFIG | 陀螺仪量程配置 |
| 0x1C | ACCEL_CONFIG | 加速度计量程配置 |
| 0x3B~0x40 | ACCEL_XOUT_H~ACCEL_ZOUT_L | 加速度计数据（6 字节，int16 大端） |
| 0x41~0x42 | TEMP_OUT_H~TEMP_OUT_L | 温度数据（2 字节） |
| 0x43~0x48 | GYRO_XOUT_H~GYRO_ZOUT_L | 陀螺仪数据（6 字节，int16 大端） |

## 五、初始化流程

### 步骤 1：I2C 外设初始化

```c
RCC_APB1PeriphClockCmd(RCC_APB1Periph_I2C1, ENABLE);
// GPIO 配置为开漏输出
// I2C 速率 400kHz（Fast Mode）
I2C_Init(I2C1, &i2c);
```

### 步骤 2：验证设备身份

```c
ret = i2c1_read_bytes(0x68, 0x75, &whoami, 1);
if (whoami != 0x68) {
    return MPU6050_ERR_ID;
}
```

### 步骤 3：唤醒传感器

```c
i2c1_write_reg(0x68, 0x6B, 0x00);
```

MPU6050 默认上电后处于 SLEEP 模式。写 0x6B = 0x00 清除 SLEEP 位（bit 6），使传感器进入工作状态。

### 步骤 4：配置采样率

```c
i2c1_write_reg(0x68, 0x19, 0x00);
```

`SMPRT_DIV` 控制采样率：`采样率 = 陀螺仪输出率 / (1 + SMPRT_DIV)`。写 0 表示最大采样率（1kHz）。

### 步骤 5：配置 DLPF（数字低通滤波器）

```c
i2c1_write_reg(0x68, 0x1A, MPU6050_DLPF_CFG_94HZ);
```

本驱动选用 94Hz 截止频率（0x02），各选项及对应的带宽和延迟：

| DLPF_CFG | 加速度计带宽 | 陀螺仪带宽 | 延迟 |
|----------|------------|-----------|------|
| 0 (260Hz) | 260Hz | 256Hz | 0.7ms |
| 1 (184Hz) | 184Hz | 188Hz | 1.9ms |
| **2 (94Hz)** | **94Hz** | **98Hz** | **2.8ms** |
| 3 (44Hz) | 44Hz | 42Hz | 4.8ms |
| 4 (21Hz) | 21Hz | 20Hz | 8.3ms |
| 5 (10Hz) | 10Hz | 10Hz | 13.4ms |
| 6 (5Hz) | 5Hz | 5Hz | 18.6ms |

四轴飞行器中 94Hz 是一个较好的平衡点，可以有效滤除高频振动噪声。

### 步骤 6：配置陀螺仪量程

```c
i2c1_write_reg(0x68, 0x1B, MPU6050_GYRO_RANGE_1000);  // 0x10
// 灵敏度 32.8 LSB/°/s
g_gyro_sensitivity = 32.8f;
```

量程选项和灵敏度对照：

| 设置值 | 量程 | 灵敏度 (LSB/°/s) |
|--------|------|-----------------|
| 0x00 | ±250°/s | 131.0 |
| 0x08 | ±500°/s | 65.5 |
| **0x10** | **±1000°/s** | **32.8** |
| 0x18 | ±2000°/s | 16.4 |

四轴飞行器推荐 ±1000°/s 或 ±2000°/s，因为飞行中旋转速度可能很快。

原始值转物理值的公式：

```c
gyro_x (°/s) = raw_value / sensitivity - bias;
```

### 步骤 7：配置加速度计量程

```c
i2c1_write_reg(0x68, 0x1C, MPU6050_ACCEL_RANGE_8G);    // 0x10
// 灵敏度 4096 LSB/g
g_acce_sensitivity = 4096.0f;
```

量程选项和灵敏度对照：

| 设置值 | 量程 | 灵敏度 (LSB/g) |
|--------|------|---------------|
| 0x00 | ±2g | 16384 |
| 0x08 | ±4g | 8192 |
| **0x10** | **±8g** | **4096** |
| 0x18 | ±16g | 2048 |

四轴飞行器推荐 ±8g，平衡了精度和量程。

原始值转物理值的公式：

```c
acce_x (g) = raw_value / sensitivity;
```

## 六、陀螺仪校准

陀螺仪存在零偏误差——即使静止不动，输出也不为零。本驱动的 `mpu6050_calibrate_gyro()` 实现：

```c
void mpu6050_calibrate_gyro(void)
{
    for (i = 0; i < 500; i++) {
        mpu6050_get_gyro(&gyro);
        sum_x += gyro.gyro_x;
        sum_y += gyro.gyro_y;
        sum_z += gyro.gyro_z;
        delay_us(2000);  // 每 2ms 采样一次，总共约 1s
    }
    // 取平均作为零偏
    g_gyro_bias_x = sum_x / 500.0f;
    g_gyro_bias_y = sum_y / 500.0f;
    g_gyro_bias_z = sum_z / 500.0f;
}
```

校准后的陀螺仪读数自动减去零偏：

```c
gyro->gyro_x = raw_x / g_gyro_sensitivity - g_gyro_bias_x;
```

**注意事项**：
- 校准必须在传感器静止时进行
- 每次上电后都需要重新校准（零偏随温度变化）
- 采样 500 次（约 1 秒）可以得到较稳定的平均零偏

## 七、数据读取

### 加速度计

```c
void mpu6050_get_acce(mpu6050_acce_value_t *acce)
{
    uint8_t buf[6];
    int16_t raw_x, raw_y, raw_z;

    i2c1_read_bytes(0x68, 0x3B, buf, 6);  // 一次 burst 读取 6 字节

    raw_x = (int16_t)((buf[0] << 8) | buf[1]);  // int16 大端
    raw_y = (int16_t)((buf[2] << 8) | buf[3]);
    raw_z = (int16_t)((buf[4] << 8) | buf[5]);

    acce->acce_x = (float)raw_x / 4096.0f;  // 除以灵敏度
    acce->acce_y = (float)raw_y / 4096.0f;
    acce->acce_z = (float)raw_z / 4096.0f;
}
```

**静止时的典型输出**：`acce_z ≈ 1.0g`，`acce_x ≈ 0`，`acce_y ≈ 0`

### 陀螺仪

```c
void mpu6050_get_gyro(mpu6050_gyro_value_t *gyro)
{
    // 与加速度计类似，但地址从 0x43 开始
    i2c1_read_bytes(0x68, 0x43, buf, 6);

    raw_x = (int16_t)((buf[0] << 8) | buf[1]);

    gyro->gyro_x = (float)raw_x / 32.8f - g_gyro_bias_x;
    // bias 在校准时计算，存储在静态变量中
}
```

**静止时的典型输出**：三个轴都接近 0（已减去零偏后）

### 温度

```c
float mpu6050_get_temp(void)
{
    raw = (int16_t)((buf[0] << 8) | buf[1]);
    return 36.53f + (float)raw / 340.0f;
}
```

温度值主要用于监测传感器工作状态，不建议作为精确温度计使用。

## 八、I2C 总线故障恢复

MPU6050 驱动中实现了 `i2c1_bus_reset()`，在 I2C 总线卡死（SCL 被从设备拉低）时恢复：

```c
static void i2c1_bus_reset(void)
{
    // 1. 将 SCL/SDA 切换为 GPIO 推挽输出
    // 2. 发送 9 个时钟脉冲（允许从设备完成当前操作释放总线）
    for (i = 0; i < 9; i++) {
        GPIO_ResetBits(..., SCL); delay_us(10);
        GPIO_SetBits(..., SCL);  delay_us(10);
    }
    // 3. 发送 STOP 条件
    // 4. 恢复为开漏模式，重新初始化 I2C
}
```

## 九、加速度计 vs 陀螺仪：互补关系

理解加速度计和陀螺仪的各自特点，是姿态解算的关键：

| 传感器 | 优点 | 缺点 |
|--------|------|------|
| **陀螺仪** | 动态响应快，短时间内精度高 | 有零偏漂移，积分后误差累积 |
| **加速度计** | 长期稳定，可提供绝对参考（重力方向） | 动态响应慢，受振动噪声影响大 |

**为什么需要融合？**
- 只用陀螺仪：积分得到的角度会随时间漂移（因为零偏和积分误差累积）
- 只用加速度计：可以测量静态倾角，但飞行器运动加速度会污染测量值
- **两者融合**：陀螺仪提供短期动态响应，加速度计提供长期参考来校正漂移

这正是 Mahony 互补滤波算法做的事情。
