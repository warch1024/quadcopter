# NRF24L01 使用详解

## 一、模块概览

NRF24L01 是 Nordic Semiconductor 推出的**单芯片 2.4GHz 射频收发器**，在四轴飞行器项目中用作**遥控器接收端**，接收来自 HU-M40 遥控器（基于 NRF24L01 的自定义协议）的摇杆/按键数据。

### 核心特性

| 参数 | 值 |
|------|-----|
| 工作频段 | 2.4GHz ~ 2.525GHz（ISM 频段） |
| 调制方式 | GFSK |
| 空中速率 | 250kbps / 1Mbps / 2Mbps（本驱动使用 **2Mbps**） |
| 通道数 | 126（1MHz 间隔） |
| 发射功率 | 0dBm / -6dBm / -12dBm / -18dBm |
| 供电电压 | 1.9V ~ 3.6V |
| 通信接口 | **SPI**（本驱动使用 SPI1，最高 9MHz） |
| 最大负载 | 32 字节（本驱动使用 16 字节） |

### 工作模式

| 模式 | 说明 |
|------|------|
| **PRX（主接收）** | 接收数据，本工程中飞控端使用此模式 |
| **PTX（主发送）** | 发送数据，遥控器端使用此模式 |
| 待机 | 晶体振荡器工作，SPI 可访问寄存器 |
| 掉电 | 最小功耗，SPI 也可访问 |

## 二、硬件连接

### SPI1 接口（本工程的接线）

```
STM32 F103            NRF24L01
  PA5 (SCK)   ─────── SCK
  PA6 (MISO)  ─────── MISO
  PA7 (MOSI)  ─────── MOSI
  PA4 (CSN)   ─────── CSN
  PB0 (CE)    ─────── CE
  3.3V        ─────── VCC
  GND         ─────── GND
```

### 引脚功能说明

| STM32 引脚 | 连接 NRF24L01 | 功能 |
|-----------|-------------|------|
| PA5 (SPI1_SCK) | SCK | SPI 时钟 |
| PA6 (SPI1_MISO) | MISO | SPI 主机输入/从机输出 |
| PA7 (SPI1_MOSI) | MOSI | SPI 主机输出/从机输入 |
| PA4 (SPI1_NSS) | CSN | SPI 片选（低电平有效，GPIO 软件控制） |
| PB0 | CE | 芯片使能（控制收发状态的切换） |

- **CSN 和 CE 均由 GPIO 软件控制**，CSN 控制 SPI 传输开始/结束，CE 控制射频收发状态
- CSN 和 CE 均为 **3.3V 逻辑电平**
- **供电必须为 3.3V**，不可接 5V（会烧毁模块）
- SPI 时钟为 9MHz（72MHz / 8），在 NRF24L01 最大支持 10MHz 范围内

## 三、SPI 通信协议

NRF24L01 的 SPI 协议非常简洁。每次访问需要先拉低 CSN，发送 **1 字节命令**，然后读写数据，最后拉高 CSN。

### SPI 时序参数

| 参数 | 本驱动配置 |
|------|-----------|
| 时钟极性 (CPOL) | 低电平空闲 |
| 时钟相位 (CPHA) | 第一个边沿采样 |
| 数据顺序 | MSB 优先 |
| 数据宽度 | 8 位 |

### 命令集

| 命令 | 字节码 | 说明 |
|------|--------|------|
| R_REGISTER | `0x00` | 读寄存器（低 5 位为寄存器地址） |
| W_REGISTER | `0x20` | 写寄存器（低 5 位为寄存器地址） |
| R_RX_PAYLOAD | `0x61` | 读取接收 FIFO 中的数据 |
| W_TX_PAYLOAD | `0xA0` | 写入发送 FIFO 中的数据 |
| FLUSH_TX | `0xE1` | 清空发送 FIFO |
| FLUSH_RX | `0xE2` | 清空接收 FIFO |
| REUSE_TX_PL | `0xE3` | 重复使用上次发送的负载 |
| NOP | `0xFF` | 空操作（用于获取 STATUS 寄存器值） |

### SPI 读写示例

**写寄存器**（以写 RF_CH = 25 为例）：

```c
CSN_L();
spi_readwrite_byte(0x20 | 0x05);  // W_REGISTER + RF_CH 地址
spi_readwrite_byte(25);           // 写入通道号
CSN_H();
```

**读寄存器**（以读 STATUS 为例）：

```c
CSN_L();
spi_readwrite_byte(0x00 | 0x07);  // R_REGISTER + STATUS 地址
value = spi_readwrite_byte(0xFF); // 发送 NOP 读取返回值
CSN_H();
```

**写多字节**（如写 5 字节地址）：

```c
CSN_L();
spi_readwrite_byte(0x20 | 0x10);  // W_REGISTER + TX_ADDR
while (len--) spi_readwrite_byte(*buf++);
CSN_H();
```

## 四、Enhanced ShockBurst 协议

NRF24L01 内置了 **Enhanced ShockBurst** 硬件协议栈，自动完成了：

1. **数据打包**：自动添加前导码、地址、CRC
2. **自动重发**：发送后若无 ACK 应答，自动重发
3. **自动应答**：收到有效数据后，硬件自动回复 ACK 包

### 本驱动的 ESB 配置

| 参数 | 寄存器 | 值 | 说明 |
|------|--------|-----|------|
| 自动应答 | EN_AA | 0x01 | 仅数据通道 0 使能自动应答 |
| 接收通道 | EN_RXADDR | 0x01 | 仅数据通道 0 开启 |
| 重发延迟 | SETUP_RETR[3:0] | `0x8` | 重发延迟 = (1 + 8) × 250μs = 2250μs |
| 重发次数 | SETUP_RETR[7:4] | `0x8` | 最多重发 8 次 |
| 地址宽度 | SETUP_AW | 0x03 | 5 字节地址（默认） |

**自动重发机制**：
- 发送端发送数据后自动切换到接收模式等待 ACK
- 若超时未收到 ACK，等待 2250μs 后重发
- 最多重试 8 次
- 超过最大重试次数后，STATUS 寄存器的 MAX_RT 位置位

## 五、寄存器详解

### 配置类寄存器

| 地址 | 名称 | 本驱动值 | 位说明 |
|------|------|---------|--------|
| 0x00 | CONFIG | 0x0F(RX) / 0x0E(TX) | bit0=PRIM_RX(1=接收)；bit1=PWR_UP(1=上电)；bit2~3=CRC使能/长度；bit4~6=中断屏蔽 |
| 0x01 | EN_AA | 0x01 | 使能数据通道 0 的自动应答 |
| 0x02 | EN_RXADDR | 0x01 | 使能数据通道 0 的接收地址 |
| 0x03 | SETUP_AW | 0x03 | 地址宽度 = 5 字节 |
| 0x04 | SETUP_RETR | 0x88 | 重发延迟 = 2250μs，重发次数 = 8 |
| 0x05 | RF_CH | 25 | 射频通道（2.4GHz + 25MHz = 2.425GHz） |
| 0x06 | RF_SETUP | 0x25 | bit0~3=速率(001=2Mbps)，bit4~6=发射功率(101=0dBm) |
| 0x1C | DYNPD | 0x00 | 动态负载长度（本驱动关闭） |
| 0x1D | FEATURE | 0x00 | 特性使能（本驱动关闭动态负载/ACK 负载） |

### 状态类寄存器

| 地址 | 名称 | 说明 |
|------|------|------|
| 0x07 | STATUS | bit5=TX_DS(发送完成)，bit6=RX_DR(收到数据)，bit4=MAX_RT(重发超限) |
| 0x08 | OBSERVE_TX | 发送观察寄存器，低 4 位=重发计数，高 4 位=丢包计数 |
| 0x09 | RPD | 接收功率检测（> -64dBm 时置位） |
| 0x17 | FIFO_STATUS | FIFO 状态，bit0=TX_FULL，bit1=TX_EMPTY，bit3=RX_FULL，bit4=RX_EMPTY |

### 地址寄存器

| 地址 | 名称 | 说明 |
|------|------|------|
| 0x0A | RX_ADDR_P0 | 数据通道 0 接收地址（5 字节） |
| 0x0B | RX_ADDR_P1 | 数据通道 1 接收地址（5 字节） |
| 0x0C~0x0F | RX_ADDR_P2~P5 | 数据通道 2~5 接收地址（仅最低字节可配） |
| 0x10 | TX_ADDR | 发送地址（5 字节） |

**地址匹配规则**：发送端的 TX_ADDR 必须等于接收端的 RX_ADDR_P0，双方才能通信。

### 接收负载长度寄存器

| 地址 | 名称 | 说明 |
|------|------|------|
| 0x11~0x16 | RX_PW_P0~P5 | 各通道接收负载长度（静态模式），本驱动统一设为 16 字节 |

## 六、初始化与检测流程

### 步骤 1：SPI 外设初始化

```c
spi1_init();
```

配置 PA5(SCK)、PA6(MISO)、PA7(MOSI)、PA4(CSN)、PB0(CE) 为 SPI 功能或 GPIO，初始化 SPI1 为主机模式，9MHz 时钟。

### 步骤 2：设备检测

```c
uint8_t NRF24L01_Check(void);
```

通过**写回读验证**方式检测 NRF24L01 是否存在：

```c
// 向 TX_ADDR 寄存器写入测试地址 "nRF24"
NRF24L01_Write_From_Buf(W_REGISTER | TX_ADDR, (uint8_t *)"nRF24", 5);
// 回读验证
NRF24L01_Read_To_Buf(R_REGISTER | TX_ADDR, rxbuf, 5);
// 比较 5 字节是否一致
```

检测成功返回 0，失败返回 1。

## 七、收发模式

### 接收模式（PRX）

```c
void NRF24L01_RX_Mode(uint8_t *rx_addr, uint8_t *tx_addr)
{
    CE_L();                                       // 先拉低 CE 配置
    _NRF24L01_Config(tx_addr);                    // 配置 ESB 参数（TX_ADDR 等）
    NRF24L01_Write_From_Buf(W_REGISTER | RX_ADDR_P0, rx_addr, 5);  // 设置接收地址
    NRF24L01_Write_Reg(W_REGISTER | CONFIG, 0x0F);  // PRIM_RX=1, PWR_UP=1, CRC=2字节
    CE_H();                                       // 拉高 CE 进入接收状态
}
```

- 接收地址 `rx_addr` 写 RX_ADDR_P0
- 发送地址 `tx_addr` 写 TX_ADDR（用于发送 ACK 包时作为源地址）
- CONFIG = `0x0F`：bit0=1（接收模式），bit1=1（上电），bit2=1（CRC 使能），bit3=1（CRC 2 字节）

### 发送模式（PTX）

```c
void NRF24L01_TX_Mode(uint8_t *rx_addr, uint8_t *tx_addr)
{
    CE_L();
    _NRF24L01_Config(tx_addr);                    // 设置 TX_ADDR
    NRF24L01_Write_From_Buf(W_REGISTER | RX_ADDR_P0, tx_addr, 5);  // 自收地址=发送地址，用于接收 ACK
    NRF24L01_Write_Reg(W_REGISTER | CONFIG, 0x0E);  // PRIM_RX=0, PWR_UP=1
    CE_H();
}
```

- 发送模式时 RX_ADDR_P0 设为与 TX_ADDR **相同**的地址——这是为了接收来自接收端的 **ACK 包**
- CONFIG = `0x0E`：bit0=0（发送模式），其余同上

### 收发模式切换对比

| 项目 | RX_Mode | TX_Mode |
|------|---------|---------|
| CONFIG bit0 (PRIM_RX) | 1 | 0 |
| RX_ADDR_P0 | 设置为 `rx_addr`（本地接收地址） | 设置为 `tx_addr`（用于接收 ACK） |
| TX_ADDR | 设置为 `tx_addr` | 设置为 `tx_addr` |
| CE | 拉高后持续接收 | 拉高后发送一次，发送完拉低 |

## 八、数据收发

### 发送数据

```c
uint8_t NRF24L01_TxPacket(uint8_t *tx_buf, uint8_t len);
```

流程：

1. 拉低 CE，准备好配置状态
2. 将数据写入 TX FIFO
3. 拉高 CE 启动发送
4. 等待约 1.5μs（CE 高电平时间），本驱动用空循环延时
5. 拉低 CE，读取 STATUS 寄存器
6. 检查状态：
   - `TX_DS`（bit5）：发送成功，收到 ACK
   - `MAX_RT`（bit4）：重发超过 8 次仍无 ACK，清空 TX FIFO
7. 清除对应中断标志位
8. 拉高 CE 恢复正常状态

**注意**：发送的数据长度 `len` 超过 `NRF24L01_PLOAD_WIDTH`（16 字节）时会被截断。

### 接收数据

```c
uint8_t NRF24L01_RxPacket(uint8_t *rx_buf);
```

流程：

1. 拉低 CE
2. 读取 STATUS 寄存器
3. 检查 `RX_DR`（bit6）：收到新数据
4. 若收到数据，从 RX FIFO 读取 `NRF24L01_PLOAD_WIDTH`（16）字节
5. 清除 RX_DR 标志位
6. 拉高 CE

**返回值**：1 表示收到新数据，0 表示无数据。

## 九、本工程实际用法（与 HU-M40 遥控器通信）

在 [communication.c](file:///e:/yueqian_projects/STM32-projects/quadcopter/system/communication.c) 中，NRF24L01 作为接收端与 HU-M40 遥控器通信，实现了**自动地址配对**功能。

### 通信流程

```
飞控上电 → NRF24L01_Init() + RX_Mode(默认地址)
               ↓
        等待遥控器数据
               ↓
     收到数据 → 解析 HU-M40 协议
               ↓
    切换 TX_Mode → 发送 ACK（内含 MAC 地址）
               ↓
    切换 RX_Mode(遥控器新地址)
               ↓
        建立连接，持续接收
```

### 地址配对细节

```
飞控端                    HU-M40 遥控器
  │                            │
  │  RX_Mode(0xCCCCCCCCCC)     │  ← 监听默认广播地址
  │◄────── 遥控器数据 ─────────│
  │                            │
  │  TX_Mode(0xCCCCCCCCCC)     │
  │  ──────── ACK(含飞控MAC) ──►│
  │                            │
  │  RX_Mode(遥控器地址)       │  ← 切换到遥控器的私有地址
  │◄────── 持续接收遥控器数据 ──│
```

1. 初始时飞控监听默认地址 `0xCCCCCCCCCC`
2. 收到遥控器数据后，回传 ACK，ACK 数据包中包含飞控自身的 MAC 地址
3. 遥控器收到 ACK 后切换到飞控的 MAC 地址发送
4. 飞控切换到遥控器的地址接收，建立稳定连接

## 十一、常见问题

### Q1：初始化检测失败

```
NRF24L01_Check() 返回非 0
```

可能原因：
- **接线检查**：SCK/MISO/MOSI/CSN/CE/VCC/GND 是否正确连接
- **电源电压**：NRF24L01 必须 **3.3V** 供电，部分模块需要 **3.3V 供电且电流足够**（不要在面包板上接长跳线）
- **SPI 配置**：检查 CPOL/CPHA 是否正确（CPOL=Low, CPHA=1Edge）
- **模块损坏**：NRF24L01 较为脆弱，5V 接入或静电可能损坏

### Q2：能检测到但收不到数据

- **地址不匹配**：双方的 TX_ADDR 和 RX_ADDR_P0 必须匹配
- **频率不同**：RF_CH 必须相同
- **速率不匹配**：RF_SETUP 中的速率设置必须一致
- **CRC 设置不同**：双方 CRC 设置需一致

### Q3：通信距离短

- 检查天线是否焊接良好（陶瓷天线模块需净空区）
- 发射功率是否设置为 0dBm（RF_SETUP bit4~6 = 101）
- 空速设置为 250kbps 可获得最远距离（本驱动使用 2Mbps 优先速度）

### Q4：SPI 通信故障

本驱动的 `spi1_readwrite_byte()` 使用查询方式等待标志位，若卡死可能是：
- SPI 时钟配置错误（CSN 拉低前需确保 SPI 已初始化）
- MISO 引脚浮空输入，无信号时可能读到随机值
