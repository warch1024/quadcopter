# Mahony 姿态解算算法详解

## 一、什么是姿态解算

姿态解算是指从 IMU（惯性测量单元，即加速度计 + 陀螺仪）的原始数据中，计算出物体在三维空间中的**朝向**（姿态）。通常用三种方式表示：

1. **欧拉角（Euler Angles）**：Roll（横滚）、Pitch（俯仰）、Yaw（偏航）——直观但存在万向锁问题
2. **旋转矩阵**：3x3 矩阵，计算稳定但参数冗余
3. **四元数（Quaternion）**：4 个参数，无万向锁，计算高效——**本驱动采用的方式**

## 二、Mahony 滤波器概述

Mahony 滤波器是由 Robert Mahony 提出的**互补滤波（Complementary Filter）姿态解算算法**。

### 核心思想

```
        ┌─────────────┐
陀螺仪 ─→│   角速度积分   │ ─→ 短期姿态 ─┐
        └─────────────┘              │
                                     ├→ 互补融合 → 最终姿态
        ┌─────────────┐              │
加速度计─→│ 重力方向参考   │ ─→ 长期校正 ─┘
        └─────────────┘
```

- **高频信任陀螺仪**：陀螺仪动态响应好，短时间内精度高
- **低频信任加速度计**：加速度计长期稳定，可提供重力方向作为绝对参考
- **互补融合**：通过 PI 控制器（比例-积分），用加速度计估计的误差来校正陀螺仪的漂移

### 与卡尔曼滤波的对比

| 特性 | Mahony 互补滤波 | 卡尔曼滤波 |
|------|---------------|-----------|
| 计算量 | 极低（10 次浮点运算） | 高（矩阵运算） |
| 内存占用 | 极小（几个变量） | 大（协方差矩阵） |
| 实现复杂度 | 简单 | 复杂 |
| 精度（STM32 场景） | 足够 | 更高但有资源压力 |
| 适用平台 | MCU（STM32 F103） | 高性能处理器 |

**STM32F103C8T6（72MHz，无 FPU）上 Mahony 是更实际的选择。**

## 三、算法数学原理

### 3.1 四元数基础

四元数是一个超复数，形式为：

```
q = q0 + q1·i + q2·j + q3·k
```

其中 `i² = j² = k² = ijk = -1`。

**用四元数表示旋转**：一个三维旋转可以用单位四元数 `q` 表示。

**四元数到欧拉角的转换**（本驱动的 `mahony_get_euler`）：

```c
roll  = atan2( 2*(q0*q1 + q2*q3), 1 - 2*(q1² + q2²) )
pitch = asin(  2*(q0*q2 - q3*q1) )
yaw   = atan2( 2*(q0*q3 + q1*q2), 1 - 2*(q2² + q3²) )
```

### 3.2 算法流程

Mahony 更新算法（`mahony_update` 函数）的数学流程：

**第 1 步：归一化加速度计读数**

```c
norm = sqrt(ax² + ay² + az²)
ax /= norm, ay /= norm, az /= norm
```

将加速度计向量转换为单位向量，只保留方向信息（重力方向）。

**第 2 步：从当前四元数估计重力方向**

将当前姿态的"理论重力方向"从机体坐标系映射到世界坐标系：

```c
vx = 2*(q1*q3 - q0*q2)    // 重力在 X 轴的分量
vy = 2*(q0*q1 + q2*q3)    // 重力在 Y 轴的分量
vz = q0² - q1² - q2² + q3²  // 重力在 Z 轴的分量
```

**第 3 步：计算误差（叉积）**

加速度计的实测重力方向（`ax, ay, az`）与四元数估计的重力方向（`vx, vy, vz`）之间的误差用**向量叉积**表示：

```c
ex = ay*vz - az*vy    // 绕 X 轴的误差
ey = az*vx - ax*vz    // 绕 Y 轴的误差
ez = ax*vy - ay*vx    // 绕 Z 轴的误差
```

叉积的大小正比于两个向量之间的角度差，方向为旋转轴方向。

**为什么这能表示误差？** 如果姿态准确，加速度计读数应该和四元数预测的重力方向完全一致（即 `(ax,ay,az) = (vx,vy,vz)`），此时叉积为零。任何偏差都表示姿态有误差。

**注意**：叉积误差只在 Z 轴（偏航）上较弱，因为加速度计无法测量绕重力轴的旋转（Yaw）。所以 Mahony 算法对 Yaw 的校正主要依赖于——实际上本驱动的代码中只有 Ki 积分项对 gx/gy 有作用，而 Kp 项对所有三个轴都起作用，但偏航方向由于没有加速度计的绝对参考，会产生漂移。

**第 4 步：PI 控制器**

比例-积分控制器用误差来校正陀螺仪读数：

```c
// 积分项（累积误差，消除稳态漂移）
ix += Ki * ex * dt * 2
iy += Ki * ey * dt * 2
// 注意：此处只有 ix 和 iy，没有 iz——这意味着偏航方向的漂移只能靠比例项

// 将积分误差补偿到陀螺仪读数
gx += ix + Kp * ex
gy += iy + Kp * ey
gz +=      Kp * ez   // 偏航只有比例校正
```

**为什么需要 PI 控制器？**

- **P（比例）**：立即响应误差，快速校正动态偏差
- **I（积分）**：累积长期误差，消除陀螺仪零偏引起的稳态漂移

**第 5 步：用校正后的角速度更新四元数**

四元数的微分方程：

```
dq/dt = 0.5 * q ⊗ ω
```

其中 ω 是校正后的角速度，⊗ 表示四元数乘法。展开后：

```c
qa = q0, qb = q1, qc = q2

q0 += (-q1*gx - q2*gy - q3*gz) * halfT
q1 += ( q0*gx + q2*gz - q3*gy) * halfT
q2 += ( q0*gy - q1*gz + q3*gx) * halfT
q3 += ( q0*gz + q1*gy - q2*gx) * halfT
```

这里的 `halfT` 是 `dt/2`（即 `real_dt * 0.5f`）。

**第 6 步：四元数归一化**

```c
norm = sqrt(q0² + q1² + q2² + q3²)
q0 /= norm, q1 /= norm, q2 /= norm, q3 /= norm
```

保持四元数为单位四元数，防止数值误差累积导致非单位化。

### 3.3 算法整体结构图

```
陀螺仪数据 (gx,gy,gz)        加速度计数据 (ax,ay,az)
        │                           │
        ▼                           ▼
   角速度(rad/s)                归一化到单位向量
        │                           │
        │      ┌─────────────────────┤
        │      │                     │
        │      ▼                     ▼
        │  估计重力方向(vx,vy,vz)   实测重力方向(ax,ay,az)
        │      │                     │
        │      └─────────┬───────────┘
        │                ▼
        │        叉积误差(ex,ey,ez)
        │                │
        │         ┌──────┴──────┐
        │         ▼             ▼
        │     比例(Kp)      积分(Ki)
        │         │             │
        │         └──────┬──────┘
        │                ▼
        │         校正角速度
        │                │
        └────────────────┤
                         ▼
                  四元数微分更新
                         │
                         ▼
                   四元数归一化
                         │
                         ▼
                   输出姿态(欧拉角/四元数)
```

## 四、参数调优

### Kp（比例增益）

- **作用**：控制加速度计对陀螺仪的校正强度
- **调大**：收敛快，但对振动敏感，姿态噪声大
- **调小**：姿态平滑，但收敛慢，漂移大
- **参考值**：2.0（本驱动测试值）

```
Kp 过大 → 姿态跟随加速度计抖动
Kp 过小 → 陀螺仪漂移不能及时校正
```

### Ki（积分增益）

- **作用**：消除陀螺仪零偏引起的稳态误差
- **调大**：快速消除稳态漂移，但可能引入低频振荡
- **调小**：漂移消除慢
- **参考值**：0.005（本驱动测试值）

```
Ki 过大 → 低频振荡（姿态来回摆动）
Ki 过小 → 长时间运行后仍存在稳态漂移
```

### 调试建议

1. **先调 Kp**：Ki = 0 时，逐渐增大 Kp 直到姿态响应快且不振荡
2. **再加 Ki**：缓慢增大 Ki 直到稳态误差被消除
3. **整机测试**：在振动环境中验证姿态稳定性，必要时降低 Kp

## 五、代码调用时序

```c
// 1. 初始化
mpu6050_init();
mpu6050_calibrate_gyro();
mahony_init(2.0f, 0.005f, 0.0025f);  // Kp, Ki, halfT(初始值)

// 2. 主循环（建议 200Hz 以上）
last_tick = xTaskGetTickCount();

while (1) {
    // 读取传感器
    mpu6050_get_acce(&acce);
    mpu6050_get_gyro(&gyro);

    // 计算实际 dt
    now_tick = xTaskGetTickCount();
    real_dt = (now_tick - last_tick) * portTICK_PERIOD_MS / 1000.0f;
    last_tick = now_tick;

    // 防止 dt 异常（第一次运行或任务被挂起过久）
    if (real_dt > 0.2f) real_dt = 0.005f;

    // 陀螺仪转弧度
    gx = gyro.gyro_x * DEG_TO_RAD;
    gy = gyro.gyro_y * DEG_TO_RAD;
    gz = gyro.gyro_z * DEG_TO_RAD;

    // Mahony 更新（halfT = dt/2）
    mahony_update(gx, gy, gz, acce.acce_x, acce.acce_y, acce.acce_z, real_dt * 0.5f);

    // 获取欧拉角
    mahony_get_euler(&euler);

    // 使用姿态...
}
```

### 关于 dt 的注意事项

- `mahony_update` 的最后一个参数是 `halfT`（即 `dt / 2`）
- `halfT` 的精度直接影响积分精度
- 推荐使用实时计算的 `real_dt`，而非固定值（vTaskDelay 不精确）
- 务必添加 `real_dt > 0.2f` 的保护判断

## 六、Yaw 漂移问题

**重要**：Mahony（以及所有仅使用加速度计+陀螺仪的算法）**无法对 Yaw（偏航角）进行长期稳定的校正**。原因是：

- 加速度计测量的是**重力方向**，只能提供 Roll 和 Pitch 的绝对参考
- 没有磁力计或 GPS 的情况下，Yaw 参考来源于初始值 + 陀螺仪积分
- 陀螺仪的零偏会导致 Yaw 随时间缓慢漂移

解决方案：
1. **加磁力计（HMC5883L / QMC5883L）**：提供地磁参考，用 Mahony 的扩展版本（Mag 融合）
2. **加 GPS**：飞行中通过航向提供 Yaw 参考
3. **接受漂移**：对于某些应用（如仅需保持水平），可以只关注 Roll/Pitch

## 七、本驱动 API 说明

```c
// 初始化：设置 Kp、Ki，重置四元数
void mahony_init(float kp, float ki, float halfT);

// 姿态更新：输入陀螺仪(rad/s)、加速度计(g)、halfT(dt/2)
void mahony_update(float gx, float gy, float gz,
                   float ax, float ay, float az, float halfT);

// 获取欧拉角（弧度）：roll/pitch/yaw
void mahony_get_euler(mahony_euler_t *euler);

// 获取四元数：q0/q1/q2/q3
void mahony_get_quat(mahony_quat_t *quat);
```

## 八、常见问题排查

| 现象 | 原因 | 解决方法 |
|------|------|---------|
| Roll/Pitch 静止时缓慢漂移 | Ki 太小或为 0 | 增大 Ki（如 0.01~0.02） |
| Roll/Pitch 振动时剧烈跳动 | Kp 太大 | 减小 Kp（如 0.5~1.0） |
| Yaw 随时间漂移 | 缺少磁力计参考 | 加磁力计或接受漂移 |
| 角度突变很大 | dt 异常（非真实间隔） | 检查 `real_dt` 计算 |
| 角度来回低频振荡 | Ki 太大 | 减小 Ki |
| MPU6050 读取失败 | I2C 通信异常 | 调用 `i2c1_bus_reset()` 恢复 |
